1. **No Guessing**: Guessing variable names, tool names, file paths, or schemas is strictly prohibited. Always read the actual source code or dynamically fetch schemas to verify before writing implementation logic.
2. **Recursive Scanning Required**: Whenever searching for files, functions, definitions, or implementations, you MUST scan recursively through all subdirectories in the repository to ensure absolutely nothing is missed.
3. **No Mock-ups or Placeholders**: Mock-ups and placeholders with fake, simulated, or hardcoded fallback content are strictly forbidden.
4. **No Deletion of Incomplete Code**: Do not remove existing code just because it appears unused or you do not understand its full context yet. Code must always be developed and completed.
5. **100% Implementation**: Everything delivered must be 100% implemented with real, functional, production-ready code. Do not leave "TODO" comments for the user to finish the logic.
